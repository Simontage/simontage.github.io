数据库和 SQL

数据库：保存有组织的数据的容器

数据库管理系统，DBMS：所有对数据库的操作由 DBMS 进行，是系统管理员与数据库文件之间的中间人

表：某种特性类型数据的结构化清单，在同一个数据库中，表名是唯一的

模式：关于数据库和表的布局以及特性的信息

列：表中的一个字段

数据类型：每个表列都有相应的数据类型，限制该列中存储的数据

行：表中的一个记录

主键：唯一标识每行的列

SQL：结构化查询语言，用来与数据库通信的语言

使用 SELECT 进行查询

检索单个列

  • SQL语句应该由“;”结束，某些 DBMS 并不强制要求
  • SQL 语句不区分大小写，良好的习惯是 SQL 关键字大写，所有表名和列名小写
  • SQL 语句中的空格会被忽略，可以将 SQL 语句写在一行，也可以写在多行

检索多列

检索所有列

使用通配符星号（*）返回表中的所有列

检索一列值并去掉重复的值（DISTINCT）

DISTINCT 关键字指示 MySQL 返回不同的值，当检索多列时，DISTINCT 会同时对多列生效

使用 ORDER BY 排序检索的数据

排序数据

通常ORDER BY子句排序为显示所选择的列，实际上也可以通过非检索的列排序数据

按多个列排序

检索三列数据，首先按照产品价格排序，然后安装产品名称排序。对于上面的例子中，仅有多各行具有相同的价格时，才会继续按照名称排序。如果所有的价格都是唯一的，那么不会按照名称排序。

指定排序方向（DESC）

数据排序默认为升序排序（A-Z），使用降序排序需要指定关键字 DESC

DESC关键字只应用到位于其前面的列名，如果要在多个列上都应用降序排列，需要对每个列指定DESC 关键字

使用 ORDER BY 关键字和 LIMIT 关键字，可以检索出一列的最大值或最小值

ORDER BY 子句的位置

ORDER BY 子句应该位于 FROM 子句之后。如果使用 LIMIT，它必须位于 ORDER BY 子句之后。

使用 WHERE 过滤数据

WHERE 子句的位置

在同时使用 ORDER BY 和 WHERE 子句时，应该让 ORDER BY 位于 WHERE 之后，否则会产生错误

使用引号

单引号用来限定字符串，如果将值与字符串类型的列进行比较，需要限定引号，与数值列进行比较不用引号。

范围值检查（BETWEEN）

BETWEEN 和 AND 连接两个值：所需范围的低端值和高端值，匹配的范围包含开始值和结束值。

空值检查（NULL）

多条件过滤（AND&OR）

AND 关键字用在 WHERE 子句中，用来指示检索满足所有给定条件的行

OR 关键字指示 MySQL 检索匹配任一条件的行
